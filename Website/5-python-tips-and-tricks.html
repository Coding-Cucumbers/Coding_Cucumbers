List & Dictionary Comprehension

I’m sure every Python developer worth their salt has heard of list comprehension, but are you utilizing it enough? I guess the more important question is - Why exactly are list comprehensions preferred over plain-old for loops? There are generally 2 reasons for this.

Firstly, it’s more readable! A for loop often takes up multiple lines of unnecessary code, code that can be written in one neat line using a list comprehension! Readability may not be on the top of your priorities as a beginner programmer for now, but trust me it’s important! As you progress in your programming career, it’s inevitable that you find yourself working on code in a team. This means sloppy and unreadable code will be a huge eyesore for you and your teammates. List comprehensions prevent that! 


    #Without List Comprehension!
    old_lst = [1,2,3,4,5]
    new_lst = []
    for element in old_lst:
        new_element = old_element * 2
        new_lst.append(new_element)

    #With List Comprehension!
    old_lst = [1,2,3,4,5]
    new_lst = [element * 2 for element in old_lst]

Just compare the 2 blocks of code! Using a list comprehension makes it look much more elegant, while doing exactly the same thing!


Secondly, it’s much faster! In terms of speed, using a list comprehension is on average twice as fast as using a for loop (when dealing with simple functions). This is huge in terms of runtime and that’s always important as a developer. The speed of your code often defines your skills as a programmer. No programmer with self-respect would want to accept slower code right?

However, there are cases where list comprehensions are limited in its applicability. Unfortunately, there is no way to break out of a list comprehension, unlike a for loop. If your code requires you to have a break condition, the “traditional” for loop will still be your better bet.


Now let’s talk about something maybe less known - Dictionary comprehensions! Now I’m certain way less people have heard of dictionary comprehensions, but they are actually pretty easy to use too! It’s basically the same syntax as a list comprehension! Let me demonstrate: 


    new_dictionary = {key:value for (key,value) in old_dictonary.items()}

Much like our beloved list comprehensions, dictionary comprehensions are much more aesthetically pleasing and fast! 


    #Without Dictionary Comprehension!
    old_dictionary = {“one” : 1, “two” : 2}
    new_dictionary = {}
    for key, value in old_dictionary.items():
        new_dictionary[key] = value * 2


    #With Dictionary Comprehension!
    old_dictionary = {“one”:1, “two:2}
    new_dictionary = {key : value * 2 for (key, value) in old_dictionary.items()}

Again, look at the difference! Doesn’t the dictionary comprehension just look so much more elegant and impressive?

Enumerate

Let’s talk about enumerate next! Sounds complicated, but it’s surprisingly super simple and very useful too! Basically, enumerates accepts an iterable as a parameter and returns an enumerate object.

This enumerate object is essentially just the same iterable but with a counter (index) for each element. We can convert it into a list object, a tuple object or even iterate over it as an enumerate object!

To convert it into a list or tuple, we simply use: 


    list(enumerate_object)
    tuple(enumerate_object)

And to iterate over it, we treat it almost like looping over a dictionary’s items: 


    for count, item in enumerate_object:
        print(count, item)
        #This will output the index of the item in an enumerate object!

Furthermore, enumerate also accepts a second parameter, which defines at which value the first element counts from (default is 0). For example: 


    fruits = [“apple”, “pear”, “mango”]
    enumerate_fruits = enumerate(fruits, 10)
    for count, item in enumerate_fruits:
        print(count, item)
        #This will output the same as before, but starting from 10, 11, …!

Without enumerate, we will need a lot more lines of code to produce the same output! Enumerate offers us a very elegant and clean way of looping through iterables with indexing!

Multiple variable assignment

Another thing that can make your Python code more palatable is assigning multiple variables in a single line! It’s super simple and surprisingly saves a lot of unnecessary lines of code as well! To do this, we simply use commas (,) to assign multiple variables: 


    a, b, c = 1, 2, 3
    #This assigns three different variables with three different values!

This again makes our code neat and faster as well! If you were to think oh how many times we declare and assign variables in Python, you can just imagine how much time we will save using multiple variables assignment! It all adds up!

It’s also possible to assign multiples variables to the same value in one line! It’ll look something like this: 


    a = b = c = 5
    #This assigns three different variabl;es with the same value!

Another interesting thing we can do is swap variable values! Using commas, we can assign the value of one variable to another variable, something like this: 


    a, b = b, a
    #This assigns the value of a to b, and b to a!

Without using multiple variable assignment, we would need to assign the values into a temporary variable that would not only add unnecessary lines of code, but also waste computer memory!

Unpacking variables

Unpacking variables is actually a very simple concept that's very similar to multiple variable assignment. It involves assigning multiple variables values from an iterable! Whether it be lists or tuples, sets or dictionaries, unpacking variables is a very efficient method of extracting extracting values into multiple variables.

Exactly like multiple variable assignment, we use commas when unpacking variables. Let me demonstrate: 


    a, b, c = [1, 2, 3]

You can do this with tuples, sets and dictionaries as well! As for dictionaries, you can unpack the variables as the dictionary's keys, values or items (as tuples). We simply use the dictionary's .keys(), .values() or .items() methods!

One more thing to take note of for unpacking variables is the * operator. Don't mistake this for the multiply operator! When unpacking variables, the * operator helps us to store multiple values into one variable! This means that if we have more values in our iterable than our variables, we can determine which variable will store the extra values! Let's see how that works! 


    a, b* = [1, 2, 3]
    #This will result in a = 1, b = [2, 3]

Zip

The Zip() function in Python is extremely underused. I'm sure most beginner Python developers won't even know of its existence, let alone use it in their own code. This is a huge mistake! The Zip() function is an extremely valuable and time-saving function that you must have in your Python arsenal.

The Zip() function helps us to combine iterables together. In simpler terms, it takes multiple iterables as arguments and returns one iterator object (NOT list!) with the elements from the iterables combined in a series of tuples! This should make things clearer: 


    fruits = ["apples", "pears", "oranges"]
    colours = ["red", "green", "orange"]
    iterator = zip(fruits, colours)
    #This will produce an iterator object that looks like this:
    #[("apples", "red"), ("pears", "green"), ("oranges", "orange")]

Now how can we manipulate this iterator object into something useful? It's fairly easy! If you want it as a list object, simply use the python function list()! 


    iterator_list = list(iterator)

You can even return it as a dictionary, with the values of the first iterable as keys and the values of the second as values! 


    iterator_dictionary = dict(iterator)

One important thing to take note of is what iterables you are passing into the Zip() function! Due to the "combining" functionality of the Zip() function, the order of your iterable elements matter. This in turn also means if you pass in iterables that are unordered (e.g. sets), the resulting iterator object will be jumbled and that's highly unlikely to be what you intended. So as a general rule of thumb, don't pass in any unordered iterables!
